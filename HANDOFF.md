# LessFriction 再設計・運営引き渡し書

**作成日:** 2026年7月11日  
**作成者:** Manus AI  
**対象:** 旧「つながる市場」Webアプリ

## 結論

旧アプリは、名称、情報設計、検索ページ構造、構造化データ、信頼性表示、デスクトップUIを全面的に見直し、**LessFriction（レスフリクション）**として再構築しました。

新しい価値提案は「商品をたくさん見せる」ことではなく、**暮らしの小さな手間を減らすための選び方を、向く人・向かない人・購入前チェックまで短時間で理解できる形にすること**です。検索エンジン向けの装飾を増やすのではなく、検索意図に答える固有ページと検証可能な情報を中心にしています。

> 暮らしの手間を、道具でひとつずつ減らす。

## 実施結果

| 領域 | 旧状態 | 実施後 |
|---|---|---|
| ブランド | 「つながる市場」。川・池・海という比喩が中心 | **LessFriction**。何を減らすサイトかを名称とコピーで明示 |
| 検索受け皿 | ホームとAboutが中心。商品詳細はモーダル | ホーム、About、4本の固有ガイドURLを静的生成 |
| 本文 | 商品カードと検証不能な実体験・評価値 | 課題、結論、向く人、向かない人、確認項目、注意点を可視本文で掲載 |
| 構造化データ | 根拠の弱いProduct、評価、価格、在庫 | WebSite、Organization、FAQPage、ItemList、Article、BreadcrumbListに整理 |
| UI | スマートフォン固定幅に近いSVG中心画面 | デスクトップ2列・モバイル1列の編集メディア型UI |
| 探索 | SVG上の商品選択、ランキング | キーワード検索と課題カテゴリ絞り込み |
| 信頼性 | 運営・広告・AI利用の説明が不十分 | 選定基準、情報確認、広告、AI利用、訂正方針をAboutで公開 |
| クロール | sitemap・robots・llms.txtは存在するが旧構造 | 実在URLだけのsitemap、新ブランドのrobots・llms.txtへ更新 |
| ブランド画像 | 旧ブランドのアイコン・共有画像 | 新しいOG画像、アプリアイコン、faviconへ統一 |

## 公開済みコンテンツ

初期版には、検索意図が異なる4本の課題解決ガイドを実装しました。

| カテゴリ | ガイド | 固有URL |
|---|---|---|
| デスク・充電 | USB-C 100W対応ケーブルの選び方 | `/items/usb-c-100w-cable` |
| 収納・片づけ | 折りたたみ収納ボックスの選び方 | `/items/folding-storage-box` |
| 持ち歩き | 軽量折りたたみ傘の選び方 | `/items/lightweight-folding-umbrella` |
| 学び・仕事術 | 生成AIの入門書を選ぶ基準 | `/items/ai-prompt-book-guide` |

各ページは、特定商品の購入体験を装うレビューではありません。現在は**選び方ガイド**として、仕様確認の観点と見送る条件を提供します。実際の購入、長期使用、写真撮影、比較計測が行われた時点で、検証可能な個別レビューへ拡張する設計です。

## SEO・検索設計の判断

Googleは、商品を大量にまとめただけのページより、洞察、独自調査、深い知識を示す高品質なレビューを評価すると説明しています。[1] また、ユーザー第一のコンテンツでは、独自情報、実質的な説明、情報源、著者・運営背景、制作方法、公開目的を明確にすることが推奨されています。[2]

このため、旧データに含まれていた検証不能な星評価、レビュー数、クリック数、使用期間、在庫、特定商品と結び付かない価格情報は削除しました。Product構造化データも、可視ページ上にない評価や価格を示さないよう撤去し、現在の編集ガイドに合うArticle中心へ変更しています。Googleも、構造化データはページ上の可視内容を正確に表し、誤解を招く情報を含めないよう求めています。[3]

競合調査では、「便利グッズ」「買ってよかったもの」領域の上位候補は、長期運営、実写、著者の使用経験、大量の記事、複数EC導線を強みにしていました。[4] [5] [6] [7] そこで、実績のない初期サイトが「100選」で正面衝突するのではなく、**何を買うかより、何の手間を減らすか**という課題起点へ絞っています。

## 公開に必要な最終設定

アプリ本体は公開可能な状態ですが、所有者情報や外部アカウントが必要な作業は未実施です。

| 優先度 | 必要作業 | 現在の状態 |
|---:|---|---|
| 1 | 本番ドメインを取得・接続 | `NEXT_PUBLIC_SITE_URL`で任意ドメインへ対応。未指定時は `https://lessfriction.jp` |
| 1 | ホスティングへデプロイ | プロジェクトはビルド済み。ホスティングアカウント未提供 |
| 1 | 問い合わせ窓口を設置 | Aboutに「準備中」と正直に表示 |
| 1 | Search Console登録・sitemap送信 | 公開URLと所有権確認が必要 |
| 2 | アクセス解析を設定 | 同意・プライバシー方針を決めてから導入 |
| 2 | アフィリエイトIDを設定 | 現在は通常のAmazon検索結果リンク。収益タグなし |
| 2 | 商標・ドメインの権利確認 | 公開検索上の直接競合は限定的だが、法的調査は未実施 |

本番公開時は、次のように実URLを渡してビルドしてください。

```bash
pnpm install
pnpm lint
NEXT_PUBLIC_SITE_URL=https://実際のドメイン pnpm build
pnpm start
```

## 90日運営計画

### 0〜30日: 信頼の土台

最初の1か月は記事数を無理に増やさず、問い合わせ窓口、Search Console、アクセス解析、訂正受付を整えます。既存4ガイドについて、メーカー公式仕様、規格団体、一次資料への参照リンクを追加し、検索表示と読了状況を観察します。

| 目標 | 判断指標 |
|---|---|
| インデックス状況を把握 | 6主要HTML URLの登録状況 |
| 検索意図のずれを把握 | 表示クエリ、CTR、平均順位 |
| ページ品質を把握 | ガイド到達率、外部リンク率、直帰ではなく次ページ遷移 |
| 信頼性を補強 | 問い合わせ・訂正受付の公開 |

### 31〜60日: ロングテール拡張

Search Consoleに現れた検索語を基に、カテゴリごとに3〜5本ずつ、合計12〜20本程度まで拡張します。候補は「充電ケーブル 100W 違い」「折りたたみ収納 前開き 選び方」「折りたたみ傘 200g 風に強い」のように、**課題 × 条件 × 選び方**で設計します。記事を増やす時も、内容を変えず日付だけ更新する運用や、大量自動生成は避けます。[2]

### 61〜90日: 一次情報と収益化

運営者が実際に購入・検証できる商品だけ、写真、測定方法、使用期間、比較対象、デメリットを追加します。アフィリエイトを導入する場合は、編集順位と報酬を分離し、該当リンクの直前とAboutで開示します。商品写真や文章は自ら権利を持つ素材を優先します。

## 運営ルール

新規ガイドは `src/lib/seed.ts` に追加します。最低限、減らしたい手間、結論、向く人、向かない人、購入前チェック、長所、注意点、参考価格帯、確認日、参考リンクを揃えてください。

次の表現は、証拠がある場合だけ使用します。

| 表現 | 必要な根拠 |
|---|---|
| 「実際に使った」 | 購入・使用事実、使用期間、対象型番 |
| 「比較した」 | 比較対象、条件、測定方法、結果 |
| 星評価・レビュー件数 | 取得元、取得日、対象商品の一意性 |
| 「最安」「在庫あり」 | 更新可能な正規データ取得方法 |
| 「専門家監修」 | 実在する監修者、資格、確認範囲、同意 |
