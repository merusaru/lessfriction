# 現状診断メモ

## 技術状態

Next.js 16.2.10 / React 19.2.4 / TypeScript 5。`npm ci` と `npm run build` は成功し、ホーム、about、robots、sitemap、llms.txt はすべて静的生成される。基礎的なビルド品質は良好。

## 検索・コンテンツ上の重大課題

1. **構造化データの正確性に問題がある。** 一般的なAmazon検索結果URLを商品ページとして扱い、サイト自身を商品のBrandに設定している。価格・在庫・評価・レビュー数が実在する特定商品へ検証可能に結び付いていないため、Productリッチリザルト用途として不適切になり得る。
2. **運営者の実体験という主張が検証不能。** seedデータに具体的な使用期間、評価数、クリック数が固定値で入っている。ユーザー提供の事実ではないなら削除または明確なデモ表記が必要。
3. **実質的なアフィリエイト導線がない。** URLはAmazonの一般検索URLで、アソシエイトタグ等が付いていない。現状は収益化できない可能性が高い。
4. **SEO実装量に対して検索意図の受け皿が薄い。** 単一ホーム画面とaboutだけで、商品別・悩み別・比較別の固有URLと十分な本文がない。メタタグやllms.txtを増やしても、検索流入の核となるページ数・独自情報・検証可能な価値が不足。
5. **FAQPage構造化データの可視本文対応が弱い。** FAQ内容が主にJSON-LD内だけで、画面上に同等のFAQ一覧が見えない。構造化データは可視コンテンツと一致させるべき。
6. **ブランドと検索語が噛み合っていない。** 「つながる市場」「川・池・海」は体験として独特だが、何を探せるサイトか初見で分かりにくく、商品発見・買ってよかった・生活改善・定番道具などの検索需要に直結しない。

## UX上の課題

デスクトップ表示でもコンテンツ幅が約390pxのスマホ固定に見え、広い余白を活かせない。主要商品情報がSVG上の絵文字だけで、商品名・比較ポイント・価格が一覧で読めない。商品詳細はボトムシートで確認できるが、検索エンジンとユーザーの双方にとって情報探索の効率が低い。「商品を置く」は管理者向け機能が公開画面に露出しており、localStorage保存のため運営データ更新にはならない。

## 改善方針

新ブランドは、生活を少し良くする定番品を「理由付きで短く選ぶ」価値に寄せる。川・池・海の可視化は主役から補助体験へ下げ、検索に強いカード一覧、カテゴリーページ、選定基準、比較軸、更新日、出典を主役にする。検証不能な実体験・評価・在庫・価格は撤去し、デモ段階では外部リンクも「検索する」導線として正直に表現する。将来アフィリエイトIDと正規商品情報が提供された時だけ収益リンクへ切り替えられる設計にする。
